AmbientLight

Extends Component
Defined in: src/lighting/ambientLight.js:1
Module: lighting
Parent Module: XEO

An AmbientLight defines an ambient light source of fixed intensity and color that affects all attached Entities equally.

  • AmbientLights are grouped, along with other light source types, within Lights components, which are attached to Entities.
  • When the Entities have PhongMaterials, AmbientLight color is multiplied by PhongMaterial ambient.
  • See Shader Inputs for the variables that AmbientLights create within xeoEngine's shaders.

Example

In this example we have

  • a PhongMaterial,
  • an AmbientLight,
  • a Lights containing the AmbientLight,
  • a Geometry that is the default box shape, and
  • an Entity attached to all of the above.

 var scene = new XEO.Scene();
var material = new XEO.PhongMaterial(scene, {
    ambient: [0.3, 0.3, 0.3],
    diffuse: [1, 1, 1],
    specular: [1.1, 1],
    shininess: 30
 });
// Within xeoEngine's lighting calculations, the AmbientLight's
 // ambient color will be multiplied by the Material's ambient color
var ambientLight = new XEO.AmbientLight(scene, {
    color: [0.7, 0.7, 0.7]
 });
var lights = new XEO.Lights(scene, {
    lights: [
        ambientLight
    ]
 });
var geometry = new XEO.Geometry(scene);  // Defaults to a 2x2x2 box
var entity = new XEO.Entity(scene, {
    lights: lights,
    material: material,
    geometry: geometry
 });

As with all components, we can observe and change properties on AmbientLights like so:

 // Attach a change listener to a property
 var handle = ambientLight.on("color",
 function(value) {
            // Property value has changed
    });

ambientLight.color = [0.6, 0.6, 0.6]; // Fires the change listener

ambientLight.off(handle); // Detach the change listener
